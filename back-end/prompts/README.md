# 提示词管理系统

本目录包含AI长篇写作助手的所有系统提示词文件。

## 目录结构

```
prompts/
├── genesis/          # 创世向导相关提示词
│   ├── concept.txt      # 核心概念生成
│   ├── protagonist.txt  # 主角设定生成
│   ├── world.txt        # 世界观生成
│   └── outline.txt      # 大纲生成
└── README.md         # 本文件
```

## 如何使用

### 查看提示词
直接打开对应的 `.txt` 文件即可查看当前的系统提示词内容。

### 自定义提示词
您可以根据自己的需求修改提示词文件:

1. **编辑文件**: 用任何文本编辑器打开 `.txt` 文件
2. **修改内容**: 调整提示词的语气、重点、示例等
3. **保存文件**: 保存后,后端会自动加载新的提示词
4. **重启服务**: 修改后需要重启后端服务才能生效

### 提示词编写建议

1. **明确角色定位**: 开头清晰说明AI扮演的角色
2. **具体任务描述**: 详细说明AI需要完成什么任务
3. **输出格式要求**: 明确指定JSON格式和字段结构
4. **提供示例**: 给出好的和坏的示例供参考
5. **注意事项**: 说明需要避免的问题

### 提示词文件命名规范

- 使用小写字母
- 单词间用下划线连接
- 使用 `.txt` 扩展名
- 文件名应清晰表达用途

例如:
- `concept.txt` - 概念生成
- `protagonist.txt` - 主角设定
- `world.txt` - 世界观构建

## 创世向导提示词说明

### concept.txt (核心概念)
用于生成小说的书名、流派和核心主题。重点关注:
- 市场定位和目标读者
- 高概念公式
- 卖点清晰度

### protagonist.txt (主角设定)
用于创建有血有肉的主角角色。重点关注:
- 主角三要素(目标、弱点、优势)
- 成长弧线
- 读者代入点

### world.txt (世界观)
用于构建逻辑自洽的世界观体系。重点关注:
- 力量体系和升级路径
- 势力架构和冲突
- 分层展开策略

### outline.txt (大纲)
用于生成完整的卷-章结构大纲。重点关注:
- 情绪波浪设计
- 伏笔与暗线规划
- 前30章的爽点密集度
- 可运营性和延展性

## 版本控制建议

如果您要大幅修改提示词,建议:
1. 先备份原文件(如 `concept.txt.backup`)
2. 进行修改
3. 测试效果
4. 如果效果不佳,可以恢复备份

## 技术说明

- 提示词文件采用UTF-8编码
- 后端通过 `llm_service.load_prompt()` 方法加载
- 文件路径相对于 `back-end/prompts/` 目录
- 支持中英文混合内容
