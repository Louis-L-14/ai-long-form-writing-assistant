- 5个核心模块

  利用 LLM（大语言模型）辅助写作长篇网文，核心在于**“建立上下文（Context）”**。LLM 的记忆是有限的，它记不住你脑子里模糊的想法，所以你必须把所有前期工作**显性化、文档化**。

  我们可以将前期工作划分为 **5 个核心模块**。这些模块是有顺序的，通常**前一个模块的输出**就是**下一个模块的输入**（Chain of Thought）。

  以下是基于 LLM 工作流的模块化清单：

  ------

  ### 模块一：创意孵化与核心定调 (The Seed)

  这是“无中生有”的阶段。你可以通过与 LLM 对话，把一个模糊的灵感变成一个可执行的商业卖点。

  - **LLM 生成内容：**
    1. **多版书名测试**：生成 10 个符合当前网文热点的书名。
    2. **一句话梗概 (Logline)**：提炼核心冲突。
    3. **核心卖点/爽点分析**：明确列出本书的 3 个核心看点。
    4. **金手指（Cheat Code）机制设计**：详细定义金手指的功能、限制、升级方式（这非常重要，防止 LLM 后期乱编能力）。
  - **输入 (Input)：**
    - **基础灵感**：你脑子里最初的想法（例如：“我想写个赛博朋克风格的修仙”）。
    - **目标风格**：严肃向/搞笑向/无敌流/凡人流。
    - **对标作品**（可选）：你喜欢哪本书的感觉，告诉 LLM。

  ------

  ### 模块二：世界观与规则设定 (The World Bible)

  这是 LLM 写作最容易崩的地方（出现逻辑矛盾）。你需要生成一份“世界观白皮书”，作为后续所有写作的**约束条件**。

  - **LLM 生成内容：**
    1. **力量体系表**：严格分级（如 Lv1-Lv9），并详细描述每个等级的表现力（例如：Lv3 能碎石，Lv5 能断河）。
    2. **地理/地图设定**：大陆板块、国家分布、特殊区域（禁地、秘境）。
    3. **势力与组织架构**：主要宗门/公会/公司的名称、信条、敌对关系。
    4. **经济与货币体系**：货币名称、购买力锚定（1金币能买啥）、稀有资源定义。
  - **输入 (Input)：**
    - **模块一的输出**（特别是金手指和题材）。
    - **特定要求**：例如“我不想要传统的宗门，我想要类似公司制度的修仙界”。

  ------

  ### 模块三：人物档案库 (Character Profile)

  LLM 很容易把配角写成流水线脸谱。你需要先生成详细的“人物卡”，每次写到该人物时，把卡片喂给 LLM。

  - **LLM 生成内容：**
    1. **主角详细档案**：外貌特征、性格关键词（如：腹黑、谨慎）、口头禅、核心动机。
    2. **主要配角/反派档案**：同上。
    3. **人物关系图谱**：谁是恩人，谁是宿敌，谁是潜在后宫/队友。
    4. **语气样本**：让 LLM 模拟该角色说几句话，确定对话风格（Speech Pattern）。
  - **输入 (Input)：**
    - **模块二的输出**（人物必须依附于世界观势力）。
    - **原型参考**：例如“反派性格参考《蝙蝠侠》里的小丑，但要更理智一点”。

  ------

  ### 模块四：剧情结构与大纲 (The Outline)

  不要让 LLM 直接写正文，先让它写大纲。这一步要分层级进行。

  - **LLM 生成内容：**
    1. **全书主线脉络**：起承转合，最终 BOSS 是谁。
    2. **分卷剧情大纲**：按“换地图”的逻辑，规划第一卷（前 20 万字）的主要事件。
    3. **细化章节大纲 (Chapter Beats)**：这是**写正文前最重要的一步**。生成第一卷前 10-20 章的详细简纲（每章 300 字左右，包含出场人物、核心冲突、伏笔）。
  - **输入 (Input)：**
    - **模块一（核心梗概）** + **模块二（世界观）** + **模块三（人物）**。
    - **节奏要求**：例如“要求第 3 章出金手指，第 5 章发生第一次打脸冲突”。

  ------

  ### 模块五：辅助素材库 (The Assets)

  这是为了提高 LLM 写作效率和丰富度，预先生成好的一堆“积木”。在正文写作卡顿时，直接复制粘贴这些素材给 LLM 让他扩写。

  - **LLM 生成内容：**
    1. **随机名称库**：人名（分男女、正邪）、地名、功法名、丹药名、武器名。
    2. **场景描写片段**：仙境、废墟、繁华街道的描写模板。
    3. **打斗招式描述**：生成基于你设定力量体系的招式描写（起手式、光效、破坏力）。
    4. **路人/炮灰生成器**：生成一堆有简单背景的 NPC，随用随取。
  - **输入 (Input)：**
    - **模块二（世界观风格）**：比如是“古风雅致”还是“克苏鲁诡异”。

  ------

  ### 总结：正式写作前你的“LLM 知识库”应包含什么？

  当你完成上述步骤后，你应该整理出一个 **Project Context（项目上下文文档）**，在每次开启新的对话框进行正文写作时，你可以根据需要把以下内容作为 **System Prompt（系统提示词）** 的一部分喂给它：

  1. **【核心设定】**：一句话梗概 + 金手指规则（模块一）。
  2. **【当前等级】**：主角目前的境界及能力范围（模块二）。
  3. **【当前人物】**：本章涉及角色的性格关键词（模块三）。
  4. **【本章大纲】**：这一章具体要发生什么剧情（模块四）。

  **建议的操作流：** 先用 AI 把这五个模块生成并整理到本地文档（如 Notion 或 Obsidian），这是你的“世界设定集”。**不要试图在一个超长对话中完成所有工作**，而是把这些设定作为**索引**，每次写新章节时，按需提取投喂。
